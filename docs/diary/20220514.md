# 再见了Ballance。此去一别，便是经年

往日，我写过不少退吧文章：[《关于最近做的傻事以及自己之后该做什么的规划》](./20200729.md)，[《关于我就此不想再开发Ballance相关程序这件事》](./20200214.md)，大多无疾而终，虎头蛇尾。但这次，是真的说再见了。此去一别，便是经年。我深知我是个情绪化的人，我非常容易在附有情绪的时候做出一些出格的决定，但这次决定是经过考虑的。对我而言，Ballance是挚爱，是无论如何都不能被舍弃的东西，是我在码代码方面生我养我的地方。我一直以来，始终抱着尽我最大努力回报它的想法来做事。但这样平静的日子在此时此刻结束了。事出皆有因，一些人或许是知道的，但我不想在这里赘述，因为这些事是使我破大防，而对Ballance的发展来说确是百益而无一害的，说到底是我自己的问题，是我自己心里过不去的一个坎，它无法被修正或者是缓解，也治不好，因此就让我自己一个人走就好了。

事实上，我昨晚情绪化的时候是想直接将我所有的代码全部封存成档案的，但是考虑到这可能是个错误决定，所以我还是坚持让我睡一觉起来再决定，当然，现在我仍然认为这有必要，但是项目太多了，一个一个手动去关闭太麻烦了。项目包括但不仅限于：制图插件，BM标准啥的。等以后Virtools文件格式解析出来了，谁还会用这个所谓的BM文件，估计大家都会用libcmo.dll来自己创建Virtools文档了吧（苦笑）。这次是真的所有项目都完全不再管理了。如果你有兴趣接手，可以联系我把工程转让给你，或者你自己开fork，我都是十分欢迎的。

Ballance吧不收废物，此前，chris把我的《Ballance现状与前瞻》又翻出来“批判”了一番。61是这么评价的：“读完了yyc的大作。yyc需要的可能是：个人影响力”，我也没有否认，回答：“大概是吧。”。而事实确实如此。我对Ballance一直秉承的是用爱发电的原则，当然大多数给Ballance搞技术的也是如此。但我并非毫无所求，我一直在追求一份卓越的工作，一份开创性的前无古人后无来者的工作，一份可以让我在Ballance历史上留下名字的工作，说白了，就是追求一份名声，而不图利。但就结果来看，我最终是什么都没有得到，我也没在任何地方留下我的名字，我似乎一直是在蹭热度，在吹嘘，没做过什么有意义的工作。BML插件火了就去做插件，联机火了就去做观察者。一件件事情做起来却又不了了之。我最终还是自己亲手葬送了所有的可能性。我做不到兼容层的写作，我做不到一夜之间分析出一整个CK类，更做不到为上古软件提供具有碰撞支持的联机。我对其原理理解不能，也不配去做这些。我是废物，Ballance吧不收废物，我应该自己把自己丢进时代的垃圾桶里面。

事实上，现实生活也埋藏着阴燃的残骸，那是对我的规劝。去年的考研没有考上，是我个人的咎由自取。然好巧不巧，今年家里附近又来了疫情，人们进入了封闭管理的流程。我们是知道的，封闭让人变得脆弱。虽然我个人影响不大，因为我许多事情都是在网络上就可以完成的。人们处于封闭的高压下，尽管我们可以用理性告诉我们这是唯一正确的道路，但是人还是会不由自主地憋出一些疾病来，人会或多或少变得憔悴，变得易怒，变得敏感。有的家人之间被分割在两个隔离地之间，互相无法相见，人们之间一些脆弱的关系，很容易升级成剑拔弩张的冲突。人们需要在一场封闭之后感到慰藉，释放在封闭过程中造就的压力，生活才能更好地进行下去。在封闭的这几个月中，我是可以感受到来自家人的语音语调的变化，变得是那么的疲劳与困倦。过年期间，表亲家中由于一些原因，导致表亲父母与子女之间发生一些非常不和谐的事情，至今萦绕在我心头，让我久久不能释怀，我担心如此的隔离会让她们之间的隔阂加剧，进而带来更坏的后果，因为本身她们就因为照顾病人而分身乏术，筋疲力尽，无处可释放压力了。我是一个以物喜，以己悲的人，我以他人的快乐为快乐，以他人的悲伤为悲伤。我不喜欢看到我身边的人总是如此愁眉苦脸，我的所作所为就是为了让我身边的人高兴起来。在这样一种情况下，急需要一场胜利，一场可以一锤定音定乾坤的胜利，来为大家加油打气，稳定局面。来自考研胜利的喜悦无疑是最好的选择。因而今年的考研背负了太多。

还有4周我就要结束我大学的4年生活，回家备考去了。4周时间实在是太短了，短到让人难以相信它可以转瞬即逝，一时之间竟没有可以前进的方向，找不到未曾探索的领域，因而选择了辞别，因为属于我的能力的任务已经玩成了。我与Ballance辞别，既有主观原因，又有客观原因。Ballance的未来应该留给那些富有才华，有进取心的年轻人，而非迂腐保守，却又做不出什么东西的旧人。在可见的未来，我大概是与Ballance分道扬镳，形同陌路了。那就祝Ballance武运昌隆，前程似锦吧。
